## 一、游戏概述

### 1.1 游戏简介

《边境守卫者》是一款奇幻题材的 2D 策略塔防游戏。玩家在固定路线的地图上，消耗金币建造并升级防御塔，阻止一波又一波的怪物突破防线。核心乐趣在于：**根据敌人特性与行进路线（含双入口与分叉），灵活搭配六种建筑（五战斗塔 + 矿场），并制定金币 / 钻石双轨升级策略。**

### 1.2 独特卖点（USP）

- **路线交互革新**：双出生点、汇合与分叉；部分敌人无视兵营或拆塔，迫使玩家调整布阵。
- **炮塔升级分支**：每种战斗塔五级成长，Lv.5 二选一终极分支（游戏内英文名如 Ranger / Siege）。
- **钻石经济**：Lv.4/Lv.5 消耗钻石；矿场产出 + 精英/BOSS 掉落 + 波次奖励。
- **敌人行为多样化**：治疗、自爆、召唤、无敌周期、拆塔、BOSS 两阶段等。
- **环境利用（计划中）**：古树缠绕、猎人陷阱等地图互动。

### 1.3 目标平台与操作

- **平台：** PC（鼠标）；后续可适配触屏。
- **操作（当前原型）：**
  - 底部栏选塔 → 点击高台建造
  - 点击已建塔 → 左下信息面板升级 / 出售 / 选分支
  - 点击路径上的敌人 → 右下信息面板显示名称、HP、护甲/魔抗等级、威胁描述
  - 兵营 → **Rally** → 在号召范围内点击地图调动士兵
  - 右上角 **Call Early (+gold)** 提前开波并获得金币奖励
  - 备波阶段在上/下**出生入口**显示 **?** 情报按钮，点击查看该路线下一波敌人
  - 胜利后右上角 **Reset** 或评星面板 **Reset** 立即重开当前关卡
  - **Pause / Esc** 暂停或继续游戏
  - 精英/BOSS 出场时顶部横幅 + 大血条；炮塔受击时 **Tower under attack!** 提示
  - 左上角 **Codex** 打开图鉴（**Towers / Enemies / Synergy** 三页；自动暂停，Close 恢复）
- **界面语言：** 游戏内 UI 为**英文**；本 GDD 为中文设计说明。

### 1.4 核心循环

```
开始关卡 → 15 秒备波（可 Call Early）→ 双入口交替出怪 → 击杀获金/钻 →
升级塔（金 Lv.2–3，钻 Lv.4–5 分支）→ 应对精英/BOSS → 10 波全胜 → Victory → 评星获钥匙 → 解锁下一地图
```

## 二、核心玩法与机制

### 2.1 战斗规则

- **防守点：** 1 个终点 `(15,6)`；普通漏怪扣 1 生命，精英 5，BOSS 10；撕裂者**偷 10 金不扣生命**。
- **可建造位置：** 仅 **高台（BuildPlatform）**；当前格林姆森林共 **21** 个高台，分布在道路两侧。
- **炮塔耐久（已实现）：** HP = **等级 × 50**（Lv.1 50 → Lv.5 250）；升级回满；自爆炎魔死亡爆炸、拆塔魔像攻击可摧毁炮塔（无出售退款）。
- **金币：** 起始 200；击杀掉落；每波结束额外 `波次×5` 金。
- **钻石：** 矿场 0.1 钻/秒；精英 1–2 钻、BOSS 8 钻；第 4 波起每波结束 +1 钻。
- **波次：** 共 10 波；备波 **15 秒** 后自动开波；**Call Early** 获得 `10 + 波次×2` 金并立即开波（无钻石奖励）。
- **波次情报（已实现）：** 备波阶段在**上/下出生入口**显示 **?** 按钮；点击弹出该路线下一波敌人列表与战术提示（不再使用顶部 Incoming 面板）。
- **暂停（已实现）：** 点击 **Pause** 或按 **Esc**；暂停时冻结战斗、波次与备波倒计时；Game Over / Victory 时不可暂停并自动恢复时间流速。
- **三星评价（已实现）：** 通关时按剩余生命评星；每获得**新**星级 +1 钥匙（累计最佳，可重复挑战刷星）。
- **钥匙解锁（已实现）：** 累计 **3 钥匙** 解锁地图二「熔岩裂谷」（场景尚未开发，胜利面板显示 Coming soon）。


| 星级  | 条件（20 生命起始） |
| --- | ----------- |
| ★   | 通关（生命 ≥ 1）  |
| ★★  | 剩余生命 ≥ 10   |
| ★★★ | 剩余生命 ≥ 18   |


- **HUD：** 左上角显示 `Keys: X · Best ★★☆`（当前关卡历史最佳）。
- **塔协同标签（已实现）：** 五战斗塔各有元素 Tag；**2.8 格内**邻塔触发协同；选中塔时在信息面板显示 Tag 与激活协同；**Codex → Synergy** 分页提供总览、四组组合说明与布阵建议（与 `TowerSynergyCatalog` 一致）。


| Tag       | 塔        | 协同组合        | 效果                                                 |
| --------- | -------- | ----------- | -------------------------------------------------- |
| Marksman  | Arrow    | + Ice       | Volley：对**减速**敌人 +25% 伤害                           |
| Ice       | Frost    | + Explosive | Shatter：Cannon 对**减速**敌人 +30% 伤害                   |
| Explosive | Cannon   | + Arcane    | Breach：Cannon 命中 -10 MR 4s；Arcane 对 MR 削弱目标 +15%   |
| Arcane    | Arcane   | + Guardian  | Ward：每个邻 Barracks +4 Arcane 伤害；每个邻 Arcane 士兵 +4 护甲 |
| Guardian  | Barracks | （见 Ward）    |                                                    |


- **Lv.5 分支补全（已实现）：** Cannon Lv.2+ **眩晕**；Thunder **减护甲**；Destroyer **偷甲并 buff 邻塔**；Void Rift **普攻 + 虚空脉冲**。

### 2.2 炮塔建造与升级

- **建造：** 底部六按钮（Arrow / Frost / Cannon / Arcane / Barracks / Mine）→ 点击空闲高台。
- **升级：** Lv.1→2→3 **金币**；Lv.3→4 **钻石**；Lv.4→5 **钻石 + 分支 A/B 二选一**。
- **出售：** 返还累计花费 **50% 金币**；钻石不退。
- **满级显示：** 信息面板标题与详情显示分支英文名（如 `Arrow (Ranger) Lv.5`）。

### 2.3 兵营号召（Rally）— 已实现


| 等级     | 号召半径（格） |
| ------ | ------- |
| Lv.1–2 | 1.8     |
| Lv.3–4 | 2.8     |
| Lv.5   | 4.0     |


流程：选中兵营 → **Rally** → 在蓝色范围内点击 → 士兵移动至该点并驻守。

### 2.4 环境互动（格林姆森林）— 已实现


| 机制       | 操作                   | 效果                                               | 限制                             |
| -------- | -------------------- | ------------------------------------------------ | ------------------------------ |
| **古树缠绕** | 点击地图上方绿色古树           | 消耗 **50 金**；在对应路段生成树根区域，敌人 **减速 80%**，持续 **4 秒** | 每棵树独立 **CD 60 秒**；冷却中树变灰       |
| **猎人陷阱** | 点击路径上**指定陷阱格**（橙色标记） | 消耗 **50 金** 放置；首个经过的敌人 **100 真伤 + 定身 5 秒**       | 同时仅 **1 个** 陷阱；触发后 **CD 60 秒** |


- 左树 `(4,10)` 作用于上路 `(5,9)` 附近；右树 `(12,10)` 作用于分叉 `(10,7)` 附近。
- 陷阱可放置格：`(8,6)` 汇合段、`(10,7)` 上分叉、`(10,5)` 下分叉；**高亮**表示可放置，**变灰**表示冷却中。
- 环境点击优先于造塔/选塔（与 Rally 类似，通过 `TowerBuildController` 转发）。

## 五、地图设计

### 5.1 地图一：格林姆森林 (Grimm Forest) — 当前可玩

**地图尺寸：** 16×12 网格  

**路线结构（已实现）：**

```
出生 A (0,9) ──东行──↓ 拐角 ──→ 汇合 (6,6)
出生 B (0,3) ──东行──↑ 拐角 ──→ ──┘
                    主干 → 分叉 (9,6)
                          ╱    ╲
                    上路      下路
                          ╲    ╱
                       合流 (12,6) → 终点 (15,6)
```

- **出生点：** 2 个（上 `(0,9)`、下 `(0,3)`，蓝色标记）

## 六、数值与平衡性

- 初始 200 金、0 钻、20 生命。
- 波次奖励：金币 `波次×5`；钻石第 4 波起每波 +1。
- Call Early：`10 + 当前波次×2` 金。
- 单塔升满约需 15–18 钻；全图 10 波预期可支撑 2–3 条满分支路线。
- **v0.9 平衡：** 低等级塔（Lv.1–2）伤害上调；Lv.4–5 分支（Ranger/Siege/Blizzard/Destroyer 等）伤害下调；精英/BOSS 生命上调。

## 七、技术要点

- **引擎：** Tuanjie 1.9.1（Unity 中国版 fork）
- **地图：** `MapGridController` + `GrimmForestMapLayout` 网格与 Waypoint；4 条路线（2 出生 × 2 分叉）
- **塔：** `TowerBase` → `CombatTowerBase` / `BarracksTower` / `DiamondMineTower`；`TowerSynergyService` 邻塔协同
- **敌人：** `EnemyBase` + `EnemyBehaviors` 组件；`WaveManager` + `GrimmForestWaves`
- **UI：** 运行时生成（`GameUiController`、`TowerInfoPanelUI`、`EnemyInfoPanelUI`、`GameHUD`、`CombatPresentationUI`、`VictoryResultUI`、`CodexMenuUI`）；英文文案；**Wave Intel 面板**；**Codex 图鉴**
- **环境：** `MapEnvironmentController`、`AncientTree`、`HunterTrap`、`RootEntangleZone`
- **进度：** `LevelProgressService` + `PlayerPrefs`（最佳星级、总钥匙数）
- **待办：** ScriptableObject 数据驱动、地图二场景
